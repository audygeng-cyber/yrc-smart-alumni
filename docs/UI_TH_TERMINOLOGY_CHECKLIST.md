# UI/API Thai Terminology Checklist

เอกสารนี้ใช้เป็นมาตรฐานคำศัพท์ฝั่ง `frontend` และข้อความ response ที่ผู้ใช้เห็นจาก `backend` เพื่อให้สอดคล้องกันทั้งระบบ โดยเฉพาะ Member / Committee / Academy portals และแผงผู้ดูแล (Admin)

## คำศัพท์มาตรฐาน

- `Admin` ใช้รูปแบบ `ผู้ดูแล (Admin)` หรือ `Admin key (x-admin-key)` ตามบริบทเทคนิค
- `LINE UID` ใช้ตัวพิมพ์ใหญ่เสมอ (ห้ามใช้ `Line UID`)
- `Academy` ในข้อความไทย ใช้ `โรงเรียนกวดวิชา (Academy)` เมื่อเป็นชื่อโมดูล/พอร์ทัล
- `P/L` คงรูปแบบเดิม (ไม่แปลย่อ)
- `CSV` และ `XLSX` ใช้ตัวพิมพ์ใหญ่เสมอ
- `snapshot` แปลเป็น `สแนปช็อต`
- `session` แปลเป็น `เซสชัน` เมื่อเป็นข้อความผู้ใช้ (เก็บ identifier โค้ดเดิมได้)
- `workflow` ใช้ `ขั้นตอนงาน (workflow)` เมื่อต้องอธิบายเชิงเทคนิค
- `journal` ใช้ `สมุดรายวัน (journal)` เมื่อต้องอธิบายเชิงบัญชี
- `migration` ใช้ `ไมเกรชัน (migration)` ในข้อความแนะนำระบบ
- `Home Screen` ใช้ `หน้าจอหลัก (Home Screen)` สำหรับ iOS/PWA context
- `Funnel` ใช้ `ขั้นตอน Funnel` หรือ `Funnel` ตามพื้นที่ที่ต้องการคงศัพท์ธุรกิจ
- ข้อความ error/validation ฝั่ง API ให้ใช้รูปแบบไทยที่คงที่ เช่น `ต้องระบุ...`, `ไม่พบ...`, `...ไม่สำเร็จ`, `ไม่ได้รับอนุญาต`

## Checklist ก่อน merge

- [ ] ป้ายเมนู/หัวข้อหลัก ไม่มีอังกฤษหลุดโดยไม่จำเป็น
- [ ] ปุ่ม/placeholder/toast ใช้คำศัพท์มาตรฐานตามรายการด้านบน
- [ ] คำว่า `LINE UID`, `CSV`, `XLSX`, `Admin key (x-admin-key)` ใช้เคสตัวอักษรถูกต้อง
- [ ] ข้อความที่อธิบายระบบ ใช้รูปแบบ `ไทย + (อังกฤษ)` เฉพาะจุดที่ช่วยลดความคลุมเครือ
- [ ] โค้ด identifier, API field, enum value ไม่ถูกแปล
- [ ] ข้อความ response ฝั่ง backend ที่ส่งถึงผู้ใช้ ใช้ถ้อยคำไทยมาตรฐานเดียวกัน

## หมายเหตุการใช้งาน

- ถ้าข้อความอยู่ใน payload ที่ backend ใช้เทียบค่า ให้คงค่าอังกฤษเดิมและแปลเฉพาะ label ฝั่งแสดงผล
- ถ้าข้อความเป็น error/debug ภายในทีม ให้เน้นชัดเจนก่อนความสวยงาม แต่คงคำศัพท์มาตรฐานหลักไว้
